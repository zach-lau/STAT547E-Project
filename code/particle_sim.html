Particle Coupling Simulator

What is being simulated?

We maintain M = 2N particles, each with a fixed x-coordinate (drawn from marginal r) and a mutable y-coordinate (drawn from marginal c). The goal is to sample a coupling — a joint distribution P(x, y) with prescribed marginals r and c — that minimises a transport cost regularised by entropy.

Regularised Optimal Transport

Classical optimal transport finds the coupling that minimises ∫ c(x,y) dP(x,y) subject to the marginal constraints. The entropy-regularised version adds a penalty −ε · H(P), where H is the entropy of P and ε > 0 is the regularisation strength. The unique solution takes the Gibbs form:

P*(x, y) = u(x) · e−c(x,y)/ε · v(y)

for scaling functions u, v determined by the marginal constraints. The Sinkhorn algorithm iteratively updates these scalings until convergence. As ε → 0, the solution concentrates on the optimal transport plan; as ε → ∞, it approaches the independent coupling r ⊗ c.

Schrödinger Bridge Problem

The entropy-regularised OT problem is equivalent to the Schrödinger bridge problem, originally posed by Erwin Schrödinger in 1931–32. Given two marginals and a reference process (here: Brownian motion), find the law of the stochastic process closest to the reference (in KL divergence) that has the prescribed marginal distributions at times t=0 and t=1.

The optimal process is a Brownian bridge mixture: for each particle pair (xᵢ, yᵢ), the path is a Brownian bridge from x_i at t=0 to y_i at t=1, with variance ε · t(1−t) at intermediate time t. The Bit-Flip MH Kernel

We use a Markov chain that preserves marginal invariance exactly. The algorithm alternates between two phases each consisting of a full bit-sweep followed by a random-pairing shuffle:

  1. y-swap phase: cycle through bit indices k = 0,…,N−1; for each k propose exchanging yᵢ ↔ yⱼ for all pairs (i, i ⊕ 2ᵏ), then run a random-pairing y-shuffle.
  2. x-swap phase: same bit-sweep structure, but propose exchanging xᵢ ↔ xⱼ, then run a random-pairing x-shuffle.

Both swap types use the same Metropolis–Hastings acceptance probability:

α = min(1, exp(−ΔE/ε))

where ΔE = (xᵢ−yⱼ)² + (xⱼ−yᵢ)² − (xᵢ−yᵢ)² − (xⱼ−yⱼ)² is the change in squared-distance cost (symmetric in x and y). Each phase only moves one set of values, so both empirical marginals are exactly preserved at every step.

The random-pairing shuffle after each bit-sweep ensures ergodicity by preventing the chain from cycling without mixing.
